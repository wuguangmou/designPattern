备忘录模式：保存一个对象的某个状态，以便在适当的时候恢复对象。
类型：行为型。备忘录模式可以理解为“后悔药”，在某些特定的应用场景中会用到。

适用场景：保存及恢复数据相关的业务场景；
          游戏时的存档，或者后悔时，想恢复到之前的状态。

优点：为用户提供了一种可恢复状态的机制；
      实现了信息的封装。
缺点：会造成资源的占用。
